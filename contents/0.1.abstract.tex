% Recently, a class of hardware side-channel vulnerabilities has been
% discovered in field-programmable gate arrays (FPGAs), in which
% semiconductor defect dynamics within reconfigurable routing resources
% leak traces of prior users' activity. Specifically, bias temperature
% instability (BTI) phenomena cause recoverable drift in transistor
% switching time under prolonged stress voltage. In FPGA routes, these
% subtle BTI-dependent propagation delay fluctuations accummulate,
% producing remnants called FPGA pentimenti. These FPGA pentimenti are
% extracted using fine-grained propagation delay sensors. The viability
% of this attack depends on sensor measurement quality, which previously
% relied on the high resolution phase shift capabilities included on
% Xilinx products. In recent work, we have shown that equivalent tuning
% resolutions can be obtained on FPGAs from other vendors by
% reformulating fine-grained sensor tuning as a constrained PLL
% configuration search. With FPGA pentimenti measurement capabilities now
% broadly accessible across many FPGA vendors, a natural next question is
% whether the underlying attack class extends beyond FPGAs.

% Resistive random-access memory (ReRAM) is a compelling candidate for
% pentimenti-like attacks beyond FPGAs. In ReRAM, data is stored as a
% defect configuration in a conductive filament and write operations are
% performed by program-verify loops whose duration is dictated by
% stochastic conductive filament dynamics. This relationship between
% defect dynamics and programming latency resembles the relationship
% between transistor defect dynamics (BTI) and propagation delay
% underlying FPGA pentimenti. However, several major challenges must
% first be overcome to evaluate commercial ReRAM for pentimenti-type
% attacks. Primarily, the observable programming latency of commercial
% ReRAM modules reflects word-level switching behavior, which is
% dominated by deterministic, data-dependent biases. These biases are
% introduced through a combination of write controller sequences which
% serialize the voltage polarity applied to cells within a word and
% additional switching activity introduced by undocumented
% error-correcting code (ECC) parity bits embedded within ReRAM arrays.

% In this dissertation, we demonstrate that the structural and stochastic
% components of commercial ReRAM write latency can be decoupled and
% recovered through non-invasive timing analysis alone. We perform the
% first systematic characterization of write latency in commercial ReRAM
% crossbar memories, identifying write controller voltage polarity
% serialization and ECC parity activity as the dominant structural
% sources of variance. Building on this characterization, we introduce
% \underline{B}it-exact \underline{R}ecovery of \underline{E}CC from
% \underline{W}rite Timing in \underline{R}eRAM \underline{C}rossbars
% (BREW-RC), the first technique to recover the bit-exact ECC parity
% submatrix of a commercial ReRAM using only write-access timing. BREW-RC
% formulates ECC recovery as a SAT problem with constraints derived from
% write latency distributions, and we validate the recovered ECC against
% an independent destructive reverse-engineering result. With the
% structural component recovered, we isolate the stochastic component
% directly. We then implement a framework to evaluate the entropy
% properties of the stochastic latency component by constructing a ReRAM
% TRNG and performing rigorous entropy analysis. The resulting ECC-aware
% TRNG achieves wear-leveled operation with min-entropy rigorously
% estimated under NIST SP 800-90B, passes NIST SP 800-22 and PractRand,
% and improves throughput 17$\times$ over prior wear- and ECC-unaware
% methods. These results provide the necessary reverse engineering and
% measurement infrastructure for future study of ReRAM pentimenti by
% isolating the component of programming latency sensitive to defect
% dynamics, and providing a framework for evaluating fluctuations in
% ReRAM entropy.

% Short
Recently, a class of non-invasive hardware side-channel attacks has been
discovered in field-programmable gate arrays (FPGAs). These attacks
extract remnants of prior users' activity that persist as transistor
defect states within reconfigurable routing resources. These remnants
are known as FPGA Pentimenti. Resistive random-access memory (ReRAM) is
a compelling candidate for pentimenti-like attacks beyond FPGAs.
However, unlike FPGAs, where sophisticated on-chip sensors capable of
detecting pentimenti have been well-studied, non-invasive pentimenti
recovery in commercial ReRAM must rely on measurements of observable
write latency. These measurements are dominated by data-dependent
structural biases that obscure any underlying defect-dynamics signal. In
this dissertation, we demonstrate that the structural and stochastic
components of commercial ReRAM write latency can be decoupled and
recovered through non-invasive timing analysis alone. Our results
provide the reverse engineering and measurement infrastructure for
future study of ReRAM pentimenti by isolating the component of
programming latency sensitive to defect dynamics.
